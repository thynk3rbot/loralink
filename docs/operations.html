⚙️ System Operations

Management of the LoRaLink PC ecosystem, including service orchestration, Docker integration, and automated startup.

Ecosystem Launch Control

LoRaLink consists of several interconnected services: the MQTT Broker, Fleet Administrator, Corporate Website, and Documentation Server. These can be managed individually or as a unified stack.

The Master Launch Script

To start the entire LoRaLink ecosystem with a single click, use the root-level batch file:

Start_Loralink.bat

This script performs the following automated sequence:

  1. Docker Check: Detects if Docker Desktop is running.
  2. Auto-Start Docker: Launches Docker Desktop if it is offline.
  3. Health Monitoring: Waits for the Docker engine to be fully responsive (heartbeat check).
  4. Service Orchestration: Launches all sub-services in separate windows.

Automated Startup (Reboot Survival)

For production deployments where the PC must resume operations after a power cycle or reboot, you can register LoRaLink as a Windows Startup Task.

⚠️ Admin Rights Required

Registering a startup task requires Administrator privileges.

  1. Navigate to the tools/ directory.
  2. Right-click Register_Startup_Task.bat and select "Run as Administrator".
  3. LoRaLink is now registered in the Windows Task Scheduler to launch automatically upon user login.

Fault Tolerance & Recovery

All LoRaLink services include built-in crash recovery loops. If a Python server or background process fails unexpectedly, the batch controller will automatically attempt to reinstantiate the process after a 5-second cooldown.

[ERROR] Server crashed. Restarting in 5s...

Advanced Connectivity & Failover

LoRaLink v1.6.0 introduces a dual-layer protocol failover mechanism to ensure mission-critical commands reach their destination even in challenging RF environments.

Protocol Failover Algorithm

When a high-priority command is queued (e.g., via the FPING command or the FAILOVER_PING task type), the system follows this logic:

  1. Binary Phase: Attempt delivery using the optimized Binary Protocol (40% range boost). Retry up to 3 times if no ACK is received.
  2. Failover Phase: If binary delivery fails after all retries, the system automatically falls back to the Legacy LoRa Text (Boosted) protocol.
  3. Legacy Attempt: A standard text "PING" is sent using the slower but more resilient legacy framing.
  4. Alerting: If both protocols fail, a "Delivery Failed" alert is logged to the fleet dashboard.

Docker Integration

The MQTT Broker (EMQX) and advanced networking tunnels run as containerized services for maximum stability and easy scaling.

  • Standard Mode: mqttdocker.yml handles the core broker.
  • Production Mode: docker-compose.production.yml provides a fully containerized stack for VPS or server deployments.

Packaging for Release

To create a portable distribution of your current LoRaLink configuration (including your custom UI themes and scripts), run:

tools\Package_Release.bat

This generates a Release.zip at the root of the project, pre-configured with the automated startup and fault-tolerance logic discussed above.
