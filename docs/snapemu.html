SnapEmu Platform & Board Compatibility

Details on extending LoRaLink functionality to Heltec's SnapEmu IoT platform and tracking compatible ESP32 hardware.

1. The SnapEmu Platform Overview

SnapEmu is an open-source IoT platform developed by Heltec Automation. It is specifically designed to facilitate the integration and management of devices via LoRaWAN and MQTT protocols.

By integrating LoRaLink with SnapEmu, fleet administrators gain access to:

  • Advanced fleet visualization and remote device management.
  • Data decoding and long-term telemetry storage.
  • Customizable data analysis web-dashboards and mobile applications.

2. Compatible Hardware & Boards

LoRaLink is natively designed for ESP32 and ESP32-S3 SoCs, ensuring full alignment with Heltec's product line and SnapEmu connectivity.

Extended Compatibility

Because LoRaLink is built on the robust ESP32-S3 architecture (featuring native USB, AI acceleration capabilities, and extreme power efficiency), the core logic can be ported to third-party modules. Boards from manufacturers like Adafruit and Waveshare that utilize the ESP32-S3 are theoretically compatible with LoRaLink and SnapEmu, provided the config.h GPIO map is adjusted for their specific physical layouts.

3. SnapEmu Integration Architecture

Deployment Methods

SnapEmu provides highly flexible deployment options allowing you to scale from lab testing to vast industrial deployments:

  • Cloud Hosted: Connect natively to the public SnapEmu platform.
  • Local / Edge Compute: Run a local SnapEmu instance via Docker on local servers, edge devices, or a Raspberry Pi.

MQTT Handshake

To proxy LoRaLink data into SnapEmu, the device's internal MQTTManager bridges to the SnapEmu instance.

// Integration happens transparently through standard MQTT
{"topic": "snapemu/telemetry/{dev_name}", "payload": {"temp": 24.5, "rssi": -92}}
🔗 Docker Deployment Note

If you choose to run SnapEmu locally alongside LoRaLink's mqttdocker.yml stack, ensure that your MQTT broker (EMQX) forwards relevant telemetry topics to the SnapEmu ingestion engine, or point the Heltec node's mqtt_srv directly to the SnapEmu host IP.
