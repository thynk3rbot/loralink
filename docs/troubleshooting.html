Troubleshooting

Common issues and solutions. Click any item to expand.

Connectivity

WiFi won't connect or credentials don't persist

Ensure you're using SETWIFI <ssid> <password> via serial or the web dashboard. If credentials don't persist after reboot, this may be a known NVS issue — try WIPECONFIG and reconfigure.

LoRa messages not received by other nodes

Verify both nodes share the same crypto key and sync word. Check RADIO command output for diagnostics. Ensure antennas are connected — operating without an antenna can damage the SX1262.

BLE scan returns no devices

BLE scanning requires the Bleak library on the PC. Install with pip install bleak. The scan runs for ~10 seconds in the background — results populate automatically.

Dashboard shows "Device Offline"

Confirm the device is on the same network as your browser. Check the serial output for the device's IP. Try http://<device-ip>/api/status directly.

Firmware

Upload fails or times out

Try a different USB cable or port. Press the BOOT button during upload if auto-reset fails. Ensure PlatformIO is targeting the correct board (heltec_wifi_lora_32_V3).

Device reboots in a loop

This is typically caused by corrupted NVS data from a previous firmware version. Connect via serial and issue WIPECONFIG, or erase flash with pio run -t erase.

Hardware

OLED display is blank

The OLED uses I2C. Verify connections and that VExt is enabled. The display initializes during the boot sequence — check serial for I2C errors.

Relay clicks but load doesn't switch

Verify wiring to the relay module. Check that the relay is rated for your load voltage/current. Use a multimeter to confirm GPIO is driving the relay input HIGH.
