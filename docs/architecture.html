System Architecture

How LoRaLink routes commands between any interface through a unified pipeline.

CommandManager

The central routing engine accepts commands from any interface and dispatches them to any target:

Serial ─┐ LoRa ───┤ ┌──▶ Local GPIO / Relay / Display BLE ────┼──▶ CommandManager ┤ WiFi ───┤ └──▶ Forward to Mesh (LoRa, ESP-NOW, BLE) ESP-NOW ┘

Protocol Routing

Every protocol speaks the same command language. Routing is prefix-based:

  • NODE2 LED ON — Reaches a specific device
  • ALL STATUS — Broadcasts to entire mesh
  • No prefix — Executes locally

Mesh Topology

  • TTL-Based Routing — Up to 3 hops
  • Seen-Message Dedup — Prevents routing loops
  • ACK + Retry — Reliable delivery with configurable retries

Master-Slave Hierarchy

The PC/WebApp acts as the Master node (source of truth), broadcasting sync frames every 5 minutes. Slave nodes execute local tasks but yield to Master commands immediately.
