Multi-Device Swarm Operations

Scaling LoRaLink from a single gateway to a network of distributed edge nodes through the Swarm Dashboard.

1. Docker Broker Orchestration (EMQX)

To securely and efficiently route telemetry between multiple devices and the centralized Fleet Administrator, LoRaLink utilizes an external MQTT broker (EMQX). This handles all "System of Systems" transport over WiFi.

Deployment Configuration (mqttdocker.yml)

version: '3.8'

services:
  emqx:
    image: emqx:latest
    container_name: loralink_emqx
    ports:
      - "1883:1883"     # Standard MQTT routing between nodes
      - "8083:8083"     # WebSockets - Required for Swarm Dashboard UI
      - "18083:18083"   # EMQX Internal Admin UI
    environment:
      - "EMQX_ALLOW_ANONYMOUS=true"
🌐 WebSockets Requirement

The Swarm WebApp utilizes Port 8083 exclusively to handle WebSocket encapsulation over HTTP. Ensure this port is open on your host firewall.

2. Node Configuration for Swarm Operations

Before an individual piece of Heltec hardware can connect to the Swarm Dashboard, it must be provisioned with its own unique identity and the address of the MQTT Broker configured in Section 1.

NVS Device Setup

You must toggle the following Non-Volatile Storage keys using the Admin interface on each device:

Namespace: KeyValueDescription
loralink:wifi_ssid"YOUR_SSID"Local network access.
loralink:mqtt_entrueBrings up the MQTT client task loop.
loralink:mqtt_srv"192.168.1.100"The IP Address of your Docker Host running EMQX.
loralink:dev_nameNode-01Unique identifier to prevent reporting collisions.

3. Operating the Swarm Dashboard

The Swarm Dashboard is an isolated UI module that runs directly off of the WebSockets feed, rendering live telemetry without needing to query individual nodes via REST APIs.

Telemetry Reception

Nodes broadcast lightweight JSON heartbeat packets to the loralink/telemetry/{dev_name} topic every 5 minutes (or as defined by sched_int_global). This maintains the dashboard's "Online/Offline" health indicators and tracks battery consumption dynamically.

Global Command Injection

Using the Sidebar "Broadcast Console", you can inject commands across the entire LoRa network. For example, broadcasting RESTART publishes the message to loralink/cmd/all, causing all connected edge modules to cleanly reboot.

4. Testing Fleet Sync

To validate your multi-device swarm setup is correctly routing and acknowledging data:

  1. Boot the Broker: Run docker-compose -f mqttdocker.yml up -d. Navigate to http://localhost:18083 (Admin/public) to confirm EMQX is alive.
  2. Boot the Swarm Dashboard: Open tools/webapp/static/mqtt_dashboard.html. Verify the Connection Status bubble switches to Green (Connected).
  3. Provision a Node: Apply WiFi and MQTT settings to a Heltec V3 device. Once the device associates with WiFi, it immediately sends a heartbeat.
  4. Verify Appearance: A card displaying the Node identifier (e.g., Node-01), Battery Level, and RSSI should instantly appear on the dashboard.
  5. Ping Test: Click the "Ping" action on the Node Card. Observe the "Live Swarm Log" on the right sidebar—you should see the Ping command transmit, followed shortly by a [MSG] Pong back from the device.
